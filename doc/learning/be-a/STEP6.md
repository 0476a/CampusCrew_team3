# BE-A STEP 6

## 이 STEP에서 왜 이걸 배우는가

마지막 단계에서는 기능이 아니라 안정성이 중요하다.  
특히 인증은 로컬에서는 잘 되는데 운영에서는 잘 안 되는 경우가 많아서 배포 관점 점검이 필요하다.

## 이번 프로젝트에서 어디에 쓰는가

- EC2 배포 1차 책임
- 운영 환경 인증 점검
- 환경 변수 확인
- 배포 후 로그인 검증

## 반드시 알아야 하는 체크리스트

- [ ] 운영 환경 설정이 로컬과 다를 수 있다는 것
- [ ] JWT 관련 값이 틀리면 로그인 전체가 깨질 수 있다는 것
- [ ] 로그를 어디서 봐야 하는지 알아야 한다는 것

## 쉬운 말로 정리

| 개념 | 쉬운 설명 | 내가 적어볼 내용 |
| --- | --- | --- |
| 운영 환경 설정 | 배포 서버에서만 필요한 설정값 |  |

| 보안 점검 | 인증 기능이 위험하게 열려 있지 않은지 다시 확인하는 것 |  |

## 이번 STEP에서 바로 확인할 것

| 항목 | 힌트 | 작성란 |
| --- | --- | --- |
| 배포 담당 범위 | EC2, PostgreSQL + pgvector, 환경 변수, 최종 점검 범위를 적는다 |  |
| JWT 관련 환경 변수 | 꼭 필요한 값만 적는다 |  |
| 로그인 점검 순서 | 어떤 순서로 확인할지 적는다 |  |
| 배포 후 먼저 볼 로그 | 어디를 가장 먼저 볼지 적는다 |  |

## 실습 힌트

- 이 프로젝트에서 배포 1차 책임은 BE-A가 맡는다.
- 로컬 성공과 운영 성공은 다르다는 전제로 점검한다.
- 데모 전에는 토큰 만료 시나리오도 확인한다.
