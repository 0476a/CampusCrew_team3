# AI STEP 2

## 이 STEP에서 왜 이걸 배우는가

AI는 성공했을 때보다 실패했을 때 더 많이 당황하게 된다.  
결과가 잘 나오는 경우만 상상하기 쉬운데, 이 단계에서는 실패했을 때 서비스가 어떻게 버틸지를 먼저 생각해야 한다.

## 이번 프로젝트에서 어디에 쓰는가

- AI 응답 파싱
- 실패 처리
- 재시도 기준

## 반드시 알아야 하는 체크리스트

- [ ] AI가 빈 응답을 줄 수도 있다는 것
- [ ] 형식이 틀린 응답이 올 수도 있다는 것
- [ ] 일반 API 에러와 AI 에러를 구분해 주는 편이 좋다는 것

## 이번 STEP에서 바로 확인할 것

| 항목 | 힌트 | 작성란 |
| --- | --- | --- |
| 빈 응답 처리 | 사용자에게 어떻게 보여 줄지 적는다 |  |
| 형식 오류 처리 | JSON이 깨졌을 때 어떻게 할지 적는다 |  |

## 막히면 이렇게 생각하면 된다

- 실패를 숨기기보다 다루는 규칙을 만드는 것이 중요하다.
- 모든 실패를 무조건 재시도하지 않는다.
- 프론트가 읽을 수 있는 실패 응답 모양이 필요하다.
