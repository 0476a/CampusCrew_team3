# AI STEP 5

## 이 STEP에서 왜 이걸 배우는가

이제 AI 기능을 실제 화면과 API에 붙이기 시작한다.  
이때 가장 중요한 것은 "그럴듯한 답"보다 "항상 읽을 수 있는 형식"과 "이상한 답이 나왔을 때의 대응"이다.

## 이번 프로젝트에서 어디에 쓰는가

- AI 기능 1차 연동
- 구조화 출력 검증
- fallback 문구 정리

## 반드시 알아야 하는 체크리스트

- [ ] 배열, 객체, 누락값 규칙을 먼저 정해야 한다는 것
- [ ] 원문에 없는 내용을 너무 많이 만들면 위험하다는 것
- [ ] 실패했을 때 사용자 안내 문구도 기능 일부라는 것

## 쉬운 말로 정리

| 개념 | 쉬운 설명 | 내가 적어볼 내용 |
| --- | --- | --- |
| 구조화 출력 | 일정한 형식으로 결과를 주는 것 |  |
| hallucination 통제 | 없는 내용을 지어내지 않게 제한하는 것 |  |
| fallback | 실패했을 때 대신 보여 줄 처리 방식 |  |

## 이번 STEP에서 바로 확인할 것

| 항목 | 힌트 | 작성란 |
| --- | --- | --- |
| 필수 필드 | 반드시 있어야 하는 값들을 적는다 |  |
| 누락값 처리 | `null`인지 빈 문자열인지 정한다 |  |
| 사용자 안내 문구 | 결과가 없을 때 어떤 문구를 보여 줄지 적는다 |  |

## 막히면 이렇게 생각하면 된다

- 자연스러운 문장보다 읽기 쉬운 형식이 우선이다.
- 누락값 규칙이 없으면 프론트와 백엔드가 계속 어긋난다.
- AI 결과는 원문과 같이 보며 확인해야 한다.
