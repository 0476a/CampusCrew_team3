# AI STEP 3

## 이 STEP에서 왜 이걸 배우는가

이 단계부터 AI 기능이 "진짜 기능처럼" 보이기 시작한다.  
모델이 알아서 잘할 거라고 기대하기 쉬운데, 실제로는 입력 예시와 출력 필드가 더 중요하다.

## 이번 프로젝트에서 어디에 쓰는가

- 액션 아이템 추출
- 공지 톤 변환
- AI API 초안

## 반드시 알아야 하는 체크리스트

- [ ] 추출 기능은 어떤 항목을 뽑을지 먼저 정해야 한다는 것
- [ ] 톤 변환은 의미를 바꾸는 것이 아니라 말투를 바꾸는 것이라는 점
- [ ] API 응답 필드를 먼저 정하면 연동이 쉬워진다는 것

## 쉬운 말로 정리

| 개념 | 쉬운 설명 | 내가 적어볼 내용 |
| --- | --- | --- |
| 정보 추출 | 긴 글에서 필요한 항목만 뽑는 것 |  |
| 톤 변환 | 내용은 유지하고 표현만 바꾸는 것 |  |
| 예시 기반 설계 | 좋은 입력과 출력 예시를 먼저 만드는 것 |  |

## 이번 STEP에서 바로 확인할 것

| 항목 | 힌트 | 작성란 |
| --- | --- | --- |
| 액션 아이템 필드 | 담당자, 할 일, 기한 등을 적는다 |  |
| 톤 변환 결과 형태 | 변환된 문장만 줄지 추가 설명도 줄지 적는다 |  |
| API 요청/응답 예시 | 한 번에 이해되는 예시를 적는다 |  |

## 막히면 이렇게 생각하면 된다

- AI가 알아서 정리해 줄 것이라고 기대하지 않는다.
- 버튼 이름과 응답 필드 이름은 연결돼야 한다.
- 예시 입력 하나가 품질을 크게 바꿀 수 있다.
