# AI STEP 4

## 이 STEP에서 왜 이걸 배우는가

요약과 Q&A는 "무엇을 보낼지"를 잘 정하는 것이 핵심이다.  
자료를 많이 주면 더 잘할 것 같다고 생각하기 쉬운데, 오히려 너무 많으면 품질과 비용이 흔들릴 수 있다.

## 이번 프로젝트에서 어디에 쓰는가

- 진행 상황 요약
- 팀 문서 Q&A
- 출처 반환

## 반드시 알아야 하는 체크리스트

- [ ] 긴 입력을 그대로 다 보내지 않아도 된다는 것
- [ ] Q&A는 답변만이 아니라 근거도 중요하다는 것
- [ ] 길이 제한이 품질 관리에도 도움이 된다는 것

## 쉬운 말로 정리

| 개념 | 쉬운 설명 | 내가 적어볼 내용 |
| --- | --- | --- |
| 요약 | 긴 내용을 짧게 핵심만 남기는 것 |  |
| 질의응답 | 질문에 맞는 내용을 찾아 답하는 것 |  |
| 컨텍스트 제한 | 너무 많은 자료를 한 번에 넣지 않는 것 |  |

## 이번 STEP에서 바로 확인할 것

| 항목 | 힌트 | 작성란 |
| --- | --- | --- |
| 요약 대상 범위 | 어떤 데이터까지 포함할지 적는다 |  |
| Q&A 근거 표시 | 출처를 어떤 형식으로 보여 줄지 적는다 |  |
| 답변 길이 제한 | 몇 문장 정도로 줄지 적는다 |  |

## 막히면 이렇게 생각하면 된다

- 많이 넣는다고 항상 더 좋아지지 않는다.
- Q&A는 "왜 그런 답이 나왔는지"를 같이 보여 주는 편이 좋다.
- 짧고 일정한 출력이 프론트 연동에 유리하다.
