# Frontend STEP 6

## 이 STEP에서 왜 이걸 배우는가

마지막 단계에서는 새 기능보다 `안 깨지는 화면`이 더 중요하다.  
로컬에서는 되는데 배포 환경에서 안 되는 문제를 줄이고, 데모 때 막히지 않게 정리해야 한다.

## 이번 프로젝트에서 어디에 쓰는가

- 최종 화면 점검
- 에러 문구 정리
- 배포 전 확인
- 데모 준비

## 반드시 알아야 하는 체크리스트

- [ ] 로컬 환경과 배포 환경이 다를 수 있다는 것
- [ ] 환경 변수가 잘못되면 API가 안 붙을 수 있다는 것
- [ ] 데모는 모든 기능보다 핵심 흐름이 중요하다는 것

## 이번 STEP에서 바로 확인할 것

| 항목 | 힌트 | 작성란 |
| --- | --- | --- |
| 데모 핵심 흐름 | 로그인부터 주요 기능까지 한 줄로 적는다 |  |
| 운영 환경 체크 | API 주소, 로그인, 목록 조회 정도를 먼저 본다 |  |
| 꼭 정리할 에러 문구 | 사용자가 자주 보게 될 문구를 적는다 |  |

## 막히면 이렇게 생각하면 된다

- 새 기능 추가보다 오류를 줄이는 것이 더 중요하다.
- 데모에서는 모든 버튼보다 핵심 사용자 흐름이 먼저다.
- 모바일 폭과 에러 화면도 마지막에 꼭 확인한다.
- 배포 자체는 BE-A가 맡고, 프론트는 배포 후 화면 검증에 집중한다.
