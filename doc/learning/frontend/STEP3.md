# Frontend STEP 3

## 이 STEP에서 왜 이걸 배우는가

이 단계부터 서비스가 "진짜 팀 프로젝트 서비스처럼" 보이기 시작한다.  
목록, 생성, 참여, 대시보드가 연결되어야 사용자가 서비스 흐름을 이해할 수 있다.

## 이번 프로젝트에서 어디에 쓰는가

- 팀 목록
- 팀 생성
- 팀 참여
- 팀 대시보드

## 반드시 알아야 하는 체크리스트

- [ ] 목록 화면과 상세 화면은 역할이 다르다는 것
- [ ] `teamId`가 있으면 특정 팀 화면으로 들어간다는 것
- [ ] 대시보드는 상세 목록이 아니라 요약 화면이라는 것

## 쉬운 말로 정리

| 개념 | 쉬운 설명 | 내가 적어볼 내용 |
| --- | --- | --- |
| 리스트 렌더링 | 같은 모양의 데이터 여러 개를 반복해서 보여 주는 것 |  |
| URL 파라미터 | 주소에 들어 있는 값으로 어떤 팀인지 구분하는 것 |  |
| 조건부 렌더링 | 데이터가 있는지 없는지에 따라 다른 UI를 보여 주는 것 |  |

## 이번 STEP에서 바로 확인할 것

| 항목 | 힌트 | 작성란 |
| --- | --- | --- |
| 팀 목록 카드에 필요한 정보 | 팀 이름, 과목명 등 최소 정보만 적는다 |  |
| 팀 생성 후 흐름 | 생성 후 어디로 이동할지 적는다 |  |
| 대시보드 요약 정보 | 최근 공지, 일정, 할 일 수 같은 값만 먼저 적는다 |  |

## 막히면 이렇게 생각하면 된다

- 팀 목록 화면과 팀 내부 화면은 레이아웃이 다를 수 있다.
- 대시보드에 모든 상세 데이터를 한 번에 넣으려고 하지 않는다.
- 팀 참여 코드는 입력 규칙을 먼저 확인한다.
