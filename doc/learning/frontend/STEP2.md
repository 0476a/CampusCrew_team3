# Frontend STEP 2

## 이 STEP에서 왜 이걸 배우는가

이제 사용자가 로그인했는지 아닌지에 따라 들어갈 수 있는 화면이 달라진다.  
`로그인은 됐는데 화면이 이상하게 보이는 경우`가 자주 생기기 때문에 흐름을 먼저 잡아야 한다.

## 이번 프로젝트에서 어디에 쓰는가

- 로그인 폼
- 회원가입 폼
- 보호 라우트
- 로그인 후 `/teams` 이동

## 반드시 알아야 하는 체크리스트

- [ ] 로그인 성공과 실패가 화면에 다르게 보여야 한다는 것
- [ ] 인증이 안 되면 들어가면 안 되는 화면이 있다는 것
- [ ] 입력값 검증과 서버 에러가 다르다는 것

## 쉬운 말로 정리

| 개념 | 쉬운 설명 | 내가 적어볼 내용 |
| --- | --- | --- |
| 인증 상태 | 지금 이 사용자가 로그인된 상태인지 아닌지 |  |
| 폼 검증 | 입력값이 비어 있거나 형식이 틀렸을 때 먼저 막는 것 |  |
| 보호 라우트 | 로그인 안 했으면 못 들어가게 막는 화면 규칙 |  |

## 이번 STEP에서 바로 확인할 것

| 항목 | 힌트 | 작성란 |
| --- | --- | --- |
| 로그인 성공 후 이동 | 보통 `/teams`로 이동한다 |  |
| 실패 메시지 | 사용자가 이해할 수 있는 문장으로 적는다 |  |
| 토큰 처리 | 어디에 저장하고 언제 꺼내 쓰는지 적는다 |  |

## 막히면 이렇게 생각하면 된다

- 입력 오류와 로그인 실패는 같은 문제가 아니다.
- 백엔드가 `401`을 주는 경우와 `403`을 주는 경우를 구분한다.
- 로그인 후 새로고침했을 때도 사용자를 다시 확인할 방법이 필요하다.
