# BE-B STEP 3

## 이 STEP에서 왜 이걸 배우는가

대시보드는 데이터를 많이 보여 주는 화면이 아니라, 핵심만 빨리 보여 주는 화면이다.  
상세 목록을 그대로 가져오려 하기 쉬운데, 이 단계에서는 "요약 응답"이라는 생각이 중요하다.

## 이번 프로젝트에서 어디에 쓰는가

- 팀 대시보드
- 최근 공지/일정/할 일 요약
- 집계 응답

## 반드시 알아야 하는 체크리스트

- [ ] 대시보드용 응답은 별도로 만들 수 있다는 것
- [ ] 개수, 최근 항목, 상태 요약은 상세 목록과 다르다는 것
- [ ] 화면이 필요한 만큼만 주는 것이 좋다는 것

## 쉬운 말로 정리

| 개념 | 쉬운 설명 | 내가 적어볼 내용 |
| --- | --- | --- |
| 집계 조회 | 여러 데이터를 모아 개수나 요약으로 보여 주는 것 |  |
| 읽기 모델 | 화면에서 보기 좋은 형태로 만든 응답 구조 |  |
| 정렬 기준 | 무엇을 먼저 보여 줄지 정하는 규칙 |  |

## 이번 STEP에서 바로 확인할 것

| 항목 | 힌트 | 작성란 |
| --- | --- | --- |
| 대시보드 카드 종류 | 어떤 정보를 보여 줄지 적는다 |  |
| 최근 항목 기준 | 몇 개까지 보여 줄지 적는다 |  |
| 정렬 기준 | 최신순, 마감 임박순 등을 적는다 |  |

## 막히면 이렇게 생각하면 된다

- 대시보드는 모든 상세 정보를 싣는 화면이 아니다.
- 화면에 바로 필요한 값만 응답에 넣는다.
- 정렬 기준을 안 정하면 프론트와 계속 어긋난다.
