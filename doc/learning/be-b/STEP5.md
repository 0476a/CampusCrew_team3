# BE-B STEP 5

## 이 STEP에서 왜 이걸 배우는가

할 일과 회의록은 데이터가 자주 바뀌고, AI와도 연결될 수 있다.  
이 단계에서는 상태값과 긴 텍스트 데이터를 어떻게 다룰지 먼저 정해야 연동이 덜 흔들린다.

## 이번 프로젝트에서 어디에 쓰는가

- 할 일 CRUD
- 할 일 상태 변경
- 회의록 CRUD
- AI 결과 저장 구조

## 반드시 알아야 하는 체크리스트

- [ ] 상태값은 문자열 하나처럼 보여도 중요한 규칙이라는 것
- [ ] 회의록 원문과 AI 결과를 분리하는 편이 안전하다는 것
- [ ] 부분 수정과 전체 수정은 다를 수 있다는 것

## 이번 STEP에서 바로 확인할 것

| 항목 | 힌트 | 작성란 |
| --- | --- | --- |
| 상태값 enum | `TODO`, `IN_PROGRESS`, `DONE`처럼 먼저 적는다 |  |

## 막히면 이렇게 생각하면 된다

- 상태값 이름은 FE, BE, AI가 모두 같이 쓰게 된다.
- 회의록 본문과 AI 추출 결과는 같은 필드에 넣지 않는다.
- 부분 수정은 편하지만 규칙이 없으면 헷갈린다.
