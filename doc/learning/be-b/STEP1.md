# BE-B STEP 1

## 이 STEP에서 왜 이걸 배우는가

BE-B는 팀 안에서 실제로 쓰이는 데이터들을 담당한다.  
처음 구조를 너무 어렵게 잡기보다 "어떤 데이터가 필요한지"를 먼저 눈에 보이게 정리하는 것이 중요하다.

## 이번 프로젝트에서 어디에 쓰는가

- 공지
- 일정
- 할 일
- 회의록

## 반드시 알아야 하는 체크리스트

- [ ] 각 기능마다 저장해야 할 데이터가 다르다는 것
- [ ] 팀과 하위 데이터가 연결된다는 것
- [ ] 화면이 필요한 응답과 DB 구조가 다를 수 있다는 것

## 쉬운 말로 정리

| 개념 | 쉬운 설명 | 내가 적어볼 내용 |
| --- | --- | --- |
| CRUD 자원 설계 | 기능별로 어떤 데이터를 다루는지 나누는 것 |  |
| 엔터티 관계 | 어떤 데이터가 어떤 데이터에 속하는지 연결하는 것 |  |
| DTO | 화면에 보여 줄 모양대로 따로 만드는 응답 형태 |  |

## 이번 STEP에서 바로 확인할 것

| 항목 | 힌트 | 작성란 |
| --- | --- | --- |
| 자원별 필수 필드 | 공지, 일정, 할 일, 회의록 각각 최소 필드만 적는다 |  |
| 팀과의 연결 | 어떤 값으로 팀을 찾는지 적는다 |  |
| 화면에 내려줄 기본 응답 | 목록에서 꼭 필요한 값만 적는다 |  |

## 실습 힌트
