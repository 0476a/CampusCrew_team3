# BE-B STEP 4

## 이 STEP에서 왜 이걸 배우는가

공지와 일정은 둘 다 CRUD처럼 보이지만, 실제로는 필요한 데이터와 검증 규칙이 다르다.  
두 기능을 비슷하게 복붙하기 쉬워서 차이를 먼저 이해해야 한다.

## 이번 프로젝트에서 어디에 쓰는가

- 공지 CRUD
- 일정 CRUD
- 목록 정렬
- 날짜 검증

## 반드시 알아야 하는 체크리스트

- [ ] 공지와 일정은 저장 필드가 다르다는 것
- [ ] 일정은 날짜 검증이 특히 중요하다는 것
- [ ] 목록용 응답과 상세용 응답이 다를 수 있다는 것

## 쉬운 말로 정리

| 개념 | 쉬운 설명 | 내가 적어볼 내용 |
| --- | --- | --- |
| CRUD | 만들기, 읽기, 수정, 삭제의 기본 흐름 |  |
| 날짜 검증 | 시작과 끝 시간의 순서가 맞는지 보는 것 |  |
| 목록 정렬 | 어떤 순서로 데이터를 보여 줄지 정하는 것 |  |

## 이번 STEP에서 바로 확인할 것

| 항목 | 힌트 | 작성란 |
| --- | --- | --- |
| 공지 필수 필드 | 제목, 본문 같은 기본값을 적는다 |  |
| 일정 필수 필드 | 시작, 종료, 장소 등 필요한 값을 적는다 |  |
| 정렬 기준 | 공지는 최신순, 일정은 시간순 등 규칙을 적는다 |  |

## 막히면 이렇게 생각하면 된다

- 공지와 일정의 공통점보다 차이점을 먼저 본다.
- 일정은 날짜 오류가 자주 나므로 검증을 먼저 적는다.
- 목록 응답은 화면이 필요한 만큼만 담는다.
