# CampusCrew PLAN

## 1. 역할 분담

구현 방법과 기술 사용 기준은 포지션별 가이드를 기준으로 한다.

| 구분 | 담당자 | 담당 범위 | 완료 기준 |
| --- | --- | --- | --- |
| 프론트엔드 | FE | 화면, 라우팅, 폼, API 연동 | 주요 화면이 API와 연결됨 |
| 백엔드 | BE-A | 인증, 팀, 공통 응답, 배포 | 로그인, 팀 기능, 백엔드 배포와 AI 서버 연동이 동작함 |
| 백엔드 | BE-B | 공지, 일정, 할 일, 회의록, 대시보드 | 팀 하위 리소스 CRUD와 대시보드가 동작함 |
| AI | AI | 액션 아이템 추출, 톤 변환, 진행 상황 요약, 팀 문서 Q&A | 4개 AI 기능이 고정된 형식으로 동작함 |
| 공통 | 팀장 | 일정 관리, 범위 통제, 데모 준비 | STEP 진행과 데모 준비가 유지됨 |
| 공통 | 전원 | 명세 검토, 통합 테스트, 발표 준비 | 최종 시나리오를 함께 검증함 |

## 2. STEP 진행표

| STEP | 목표 | 프론트엔드 산출물 | BE-A 산출물 | BE-B 산출물 | AI 산출물 | 종료 조건 |
| --- | --- | --- | --- | --- | --- | --- |
| STEP 1 | 프로젝트 시작을 위한 베이스라인 학습과 스타트 준비 | 화면 구조 읽기와 라우트 이해 | 인증/팀 흐름 이해와 서버 구조 읽기 | 리소스/ERD 구조 이해 | AI 기능 구조와 입출력 흐름 이해 | 팀이 같은 기준으로 프로젝트를 시작할 수 있음 |
| STEP 2 | 인증과 기본 구조 구현 및 서버 초기 셋팅 | 로그인/회원가입/보호 라우트와 로컬 통신 기반 구현 | 인증 API, JWT, 서버 초기 셋팅 | 공통 예외와 DTO 초안 | 프롬프트/실패 규칙 초안 | 로그인 후 `/teams` 진입 가능 |
| STEP 3 | 팀 기능 구현 | 팀 목록/생성/참여/대시보드 | 팀 API | 대시보드 보조 조회 | 액션 아이템/톤 변환 초안 | 팀 생성 후 대시보드 진입 가능 |
| STEP 4 | 공지와 일정 구현 | 공지/일정 화면 | 팀 권한 검증 | 공지/일정 CRUD | 요약/Q&A 초안 | 공지와 일정 흐름 동작 |
| STEP 5 | 할 일과 회의록 구현 | 할 일/회의록 화면 | 상태 변경/작성자 검증 | 할 일/회의록 CRUD | AI 기능 1차 연동 | 핵심 기능 연결 완료 |
| STEP 6 | 통합 마감 및 배포 | UI 정리와 최종 점검 | 인증/팀/배포 점검 | 대시보드/리소스 최종 점검 | AI 최종 점검 | 전체 데모 가능 |

## 3. STEP별 체크리스트

### STEP 1
- [ ] 핵심 문서 흐름을 이해했다.
- [ ] 내 포지션의 시작 학습을 완료했다.
- [ ] 로컬에서 프로젝트를 시작할 준비가 되었다.
- [ ] 팀과 첫 구현 전에 맞춰야 할 문서 위치를 알고 있다.

### STEP 2
- [ ] 로그인과 회원가입 흐름이 동작한다.
- [ ] JWT 인증 구조가 동작한다.
- [ ] 서버 초기 셋팅을 완료했다.
- [ ] 공통 예외와 DTO 초안을 정리했다.
- [ ] 인증 연동을 확인했다.

### STEP 3
- [ ] 팀 목록, 생성, 참여 흐름이 동작한다.
- [ ] 팀 대시보드 1차 연동을 확인했다.
- [ ] 액션 아이템 추출과 톤 변환 초안을 정리했다.

### STEP 4
- [ ] 공지와 일정 흐름이 동작한다.
- [ ] 팀 권한 검증을 확인했다.
- [ ] 진행 상황 요약과 Q&A 초안을 정리했다.

### STEP 5
- [ ] 할 일과 회의록 흐름이 동작한다.
- [ ] 상태 변경과 작성자 검증을 확인했다.
- [ ] AI 기능 1차 연동을 확인했다.

### STEP 6
- [ ] 통합 테스트를 완료했다.
- [ ] 핵심 오류를 수정했다.
- [ ] 백엔드 배포, GCP AI 서버, 최종 데모를 확인했다.
