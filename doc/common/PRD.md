# CampusCrew 기획서

## 1. 문서 개요

이 문서는 프론트 1명, 백엔드 2명, AI 1명 구성을 기준으로 `CampusCrew` 프로젝트의 요구사항과 범위를 정의한다.

## 2. CampusCrew 기획

### 2.1 서비스 한 줄 정의
CampusCrew는 대학생 팀 프로젝트를 위한 소규모 협업 워크스페이스다. 팀을 만들고, 팀원과 함께 일정, 할 일, 공지, 회의록을 한곳에서 관리한다.

### 2.2 해결하려는 문제
팀 프로젝트는 메신저, 메모 앱, 일정 앱, 게시판, 문서 도구가 분산되어 있어 정보가 흩어지기 쉽다. 과제 마감일, 회의 일정, 역할 분담, 결정사항이 여기저기 흩어져 팀원이 따라가기 어렵다.

### 2.3 사용자

#### 1) 팀장
- 팀 생성
- 팀원 초대
- 공지 작성
- 할 일 배정
- 일정 등록

#### 2) 팀원
- 공지 확인
- 본인 할 일 확인 및 상태 변경
- 일정 확인
- 회의록 열람 및 작성 보조

### 2.4 핵심 시나리오

#### 시나리오 1. 팀 생성부터 첫 협업 시작까지
1. 사용자가 회원가입 후 로그인한다.
2. 새 팀을 생성한다.
3. 팀 설명과 과목명을 입력한다.
4. 팀원이 참여 코드로 들어온다.
5. 팀 공간의 대시보드가 생성된다.

#### 시나리오 2. 팀장이 일정을 잡고 할 일을 배정한다
1. 팀장이 회의 일정을 등록한다.
2. 팀장이 공지를 작성한다.
3. 팀장이 할 일을 생성하고 담당자와 마감일을 지정한다.
4. 팀원은 대시보드에서 본인 업무와 다가오는 일정을 확인한다.

#### 시나리오 3. 회의 이후 기록을 남긴다
1. 팀원이 회의록을 작성한다.
2. 다음 액션 아이템을 할 일로 옮긴다.
3. 다음 회의 일정과 연계해 관리한다.

### 2.5 핵심 기능

- 회원가입, 로그인, 로그아웃
- 팀 생성, 팀 목록 조회, 팀 상세 조회
- 팀 참여 코드 기반 팀 가입
- 공지 CRUD
- 일정 CRUD
- 할 일 CRUD 및 담당자 지정, 상태 변경
- 회의록 CRUD
- 대시보드에서 다가오는 일정, 최근 공지, 내 할 일 조회
- AI 기능 4개
  - 회의록에서 액션 아이템 추출
  - 문서 톤 변환
  - 진행 상황 요약
  - 팀 문서 Q&A

## 3. 서비스 구조

### 3.1 정보 구조
- 인증 영역
  - 로그인
  - 회원가입
- 메인 영역
  - 내 팀 목록
  - 팀 생성
- 팀 공간
  - 팀 대시보드
  - 공지
  - 일정
  - 할 일
  - 회의록
  - 팀 정보

### 3.2 화면 흐름 원칙
- 로그인 후 첫 화면은 내 팀 목록이다.
- 팀을 선택하면 팀 대시보드로 진입한다.
- 대시보드는 다른 하위 메뉴의 요약 진입점 역할을 한다.
- 작성 기능은 모두 별도 페이지로 구현한다.
